The objective of this technique is to show how to tag a hierarchical list. 

This expresses a semantic relationship of a tree view: all elements are bound together into one hierarchical list. It emphasizes that all elements are different branches of this one tree. This can be used for taxonomies or organigrams.

Recommended assistive technology [AT](https://pdfa.org/glossary-of-accessibility-terminology-in-pdf/#assitive-technology) behavior: When reading the first **L** in a List, all direct children of that **L** should be exposed. Lists contained in **LBody** tags should not be exposed until the **LBody** tag is read.

See also Multi-level lists (L_05) for other nested list techniques.
